## 프로젝트 초기화 폴더 구조 설계
- 프로젝트 루트 폴더 생성 (astrology-app)
- Backend 폴더 구성: `server/` (Node.js + Express + nodemon)
- Frontend 폴더 구성: `client/` (React + TypeScript + Tailwind CSS)
- 환경 변수 설정: `.env`
## 백엔드 구축
- Express 서버 연동: 기본 포트 및 미들 웨어(CORS, JSON 파서) 설정
- Gemini SDK 연동: `@google/generative-ai` 라이브러리 설치 및 초기화
- 운세 생성 API 엔드포인트 구현
  - `POST /api/fortune`: 사용자 데이터(생년월일, 성별, 운세 타입) 수신 및 Gemini 호출
## 프론트엔드(React) UI 개발
- 기본 레이아웃 구성: 헤더, 푸터, 메인 콘텐츠 영역
- 사용자 입력 폼 컴포넌트:
  - 생년월일 선택기, 성별 라디오 버튼
- 운세 옵션 선택 센선:
  - 기본값: 신년 운세
  - 드롭다운/리스트: 재물운, 연애운, 건강운 등
  - 커스텀 입력창: "오늘 소개팅 결과 알려줘" 등 직접 입력 가능
- 로딩 및 결과 표시 컴포넌트: 애니메이션 효과를 포함한 운세 결과창
## 프롬프트 엔지니어링(Logic)
- 시스템 프롬프트 설계: "너는 20년 경력의 베테랑 점성술사야" 형태의 역할 부여
- 데이터 파싱 로직: 입력된 생년월일을 사주나 별자리 정보로 변환하여 Gemini에게 전달하는 방식 정의
- 출력 형식 제어: 결과를 JSON이나 특정 Markdown 형식으로 받도록 유도
## 연동 및 고도화
- Axios를 이용한 통신: 프론트엔드에서 백엔드 API 호출 및 데이터 수신
- 에러 핸들링: API 할당량 초과나 네트워크 오류 시 사용자 알림
- 디자인 고도화: 신비로운 분위기의 테마 적용 (다크 모드, 별 이미지 등)

## 에러 로그
- localhost:5000 접속 시 403 에러
다른 프로그램이 이미 5000을 사용중이고, 그 프로그램이 권한 오류를 내고 있는 중.
포트를 3000으로 바꾸니 해결되었다.
